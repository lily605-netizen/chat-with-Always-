# 部署到 Vercel 步骤（手机操作版）

你的仓库文件目前是摊平放在根目录的（没有 src 文件夹），这份配置已经适配这种结构。

## 这次新增：DeepSeek 后端代理
DeepSeek 官方接口不允许网页浏览器直接调用它（会被浏览器拦截，报"网络出问题"），所以加了一个小小的后端中转文件 `deepseek.js`，前端先发给它，它再去问 DeepSeek。Anthropic 那边不受影响，不用管。

**这个文件位置比较特殊，必须放在 `api/deepseek.js` 这个路径下（在一个叫 `api` 的文件夹里面），不能跟其他文件一样摊平在根目录。**

### 怎么在 GitHub 手机网页上创建这个带文件夹的文件
1. 进入你的仓库主页
2. 点右上角 "···" 或者找 "Add file" → "Create new file" 按钮
3. 在文件名那一栏，直接输入 `api/deepseek.js`（注意带上 `api/` 这个前缀）——GitHub 会自动帮你建好 `api` 这个文件夹，不用你额外操作
4. 打开这个 zip 包里的 `deepseek.js`，全选复制内容
5. 粘贴到 GitHub 编辑框里
6. 滑到底部，点 "Commit changes"

## 其他文件改动
- `App.jsx`：里面调用 DeepSeek 的部分，从直连改成了走 `/api/deepseek`

## 怎么把改动过的文件传到 GitHub
1. 依次点开每个文件（`App.jsx` 有改动），点右上角铅笔图标进入编辑
2. 把里面的内容，替换成这个 zip 包里对应文件的内容（打开 zip 里的文件，全选复制，粘贴过去覆盖）
3. 每改完一个文件，滑到底部点 "Commit changes" 保存

## 改完之后
Vercel 会自动检测到 GitHub 有更新，重新构建。Vercel 会自动识别出 `api` 文件夹，把 `deepseek.js` 变成一个真正能用的后端接口，不需要你额外配置。等 1-2 分钟，去 Vercel 后台看部署状态。

## 以后想改代码怎么办
以后每次让 Claude 改完代码，把新文件下载下来，用同样的方式（点文件、编辑、粘贴、commit）替换到 GitHub 上就行，Vercel 会自动重新部署。

## 关于网络
国内访问 Vercel 目前大概率还是需要开代理，这个跟部署没关系，是网络线路的问题。
